# 🩺 Smart Preventive Healthcare System AI

> An AI-powered preventive healthcare web application that enables users to perform **preliminary disease prediction based on selected symptoms** using Machine Learning. The application provides secure user authentication, AI-assisted disease prediction, and a modern, responsive user interface built with React and Tailwind CSS.

> **Disclaimer:** This application is intended for educational and research purposes only. The predictions generated by the AI model are preliminary and should never replace professional medical advice, diagnosis, or treatment.

---

## 🚀 Features

### 👤 User Features
- 🔐 Secure User Registration
- 🔑 User Login Authentication
- 🔒 JWT-Based Protected Routes
- 🔐 Password Encryption using Bcrypt
- 🩺 AI Symptom Checker
- 🤖 Machine Learning Disease Prediction (Random Forest)
- 📊 Top 3 Predicted Diseases
- 📈 Confidence-Based Prediction Results
- 🧠 Symptom Pattern Analysis
- 💡 Health Guidance Messages (triage: routine / consult / urgent)
- ⚠️ Red-flag Symptom Detection (chest pain, breathlessness, etc.)
- 🎨 Modern Responsive UI (React + Tailwind CSS)
- ⚡ Fast REST API Integration (Flask backend)

## 📥 Clone Repository

```bash
git clone https://github.com/YOUR_USERNAME/digital-health-assistant.git
cd digital-health-assistant
```

*(Replace `YOUR_USERNAME` and repo name with your actual GitHub details.)*

---

## 🖥️ Backend Setup

### 1. Navigate to Backend

```bash
cd backend
```

### 2. Create Virtual Environment

```bash
python -m venv venv
```

### 3. Activate Virtual Environment (Windows)

```bash
venv\Scripts\activate
```

### 4. Install Dependencies

```bash
pip install -r requirements.txt
```

### 5. Configure Environment

Create a `.env` file in `backend/`:

```env
MONGO_URI=your_mongodb_uri_here
DB_NAME=healthcare_db
JWT_SECRET=change-me-in-env
```

> Never commit your `.env` file to GitHub. It should be ignored via `.gitignore`.

### 6. Run Backend Server

```bash
python app.py
```

Backend will start at:

```text
http://127.0.0.1:5000
```

---

## 🌐 Frontend Setup

Open a new terminal while backend keeps running:

## 🌐 REST API Endpoints

### Health & Root

| Method | Endpoint   | Description                  |
|--------|------------|------------------------------|
| GET    | `/`        | API root status              |
| GET    | `/health`  | Backend + model health check |

### Authentication

| Method | Endpoint    | Description           |
|--------|-------------|-----------------------|
| POST   | `/register` | Register a new user   |
| POST   | `/login`    | User login            |
| GET    | `/profile`  | Fetch logged-in user  |

### Symptoms & Prediction

| Method | Endpoint                | Description                           |
|--------|-------------------------|---------------------------------------|
| GET    | `/symptoms`            | List all symptoms                     |
| GET    | `/symptoms/search`     | Search symptoms by text query         |
| GET    | `/symptoms/categories` | Symptoms categorized by body system   |
| GET    | `/symptoms/alphabetical` | Symptoms grouped alphabetically      |
| POST   | `/predict`             | Predict disease from selected symptoms |

Request body example for `/predict`:

```json
{
  "symptoms": ["fever", "headache", "vomiting"]
}
```
